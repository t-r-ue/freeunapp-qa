### Planner Response

You are absolutely right. The moment we talk about a floating overlay that stays on top of other windows (like IDEs, folders, or PDF readers) with adjustable transparency, we step out of standard web browser capabilities. 

Browsers have strict security boundaries that prevent a standard website tab from floating "Always on Top" or adjusting window opacity.

Here are the paths we can take:

---

### Option A: The Chrome Extension (Sidebar / Floating Overlay)
Instead of a desktop app, we build a Chrome Extension.
* **How it works**: It adds a custom side panel (using Chrome's native Side Panel API) or injects a clean, floating iframe overlay on any tab you are testing.
* **Why it's cool**: 
  * You can inject a slider to change the opacity (transparency) of the overlay on the fly.
  * **Bonus**: We can use Chrome's extension API to automatically capture a screenshot of your current tab and attach it to the new issue instantly! No manual uploading.
* **Limitation**: It only floats over browser pages (won't float over your native file explorer, Excel, or Word).

### Option B: The Desktop App Wrapper (Electron / Tauri)
We wrap our existing Express + SQLite app into a lightweight desktop shell.
* **How it works**: We package the HTML and backend into a single executable.
* **Why it's cool**:
  * We can natively set `alwaysOnTop: true` and control window transparency down to the OS level using a settings slider in the UI.
  * You can trigger a global hotkey (e.g., `Ctrl + Shift + Q`) to toggle the window in and out of view while working in code.
* **Limitation**: It becomes a native desktop application instead of a link you can just share, and requires compiling/installing.

### Option C: The Draggable Bookmarklet (Web-only Widget)
A simple snippet of JavaScript that you save as a browser bookmark.
* **How it works**: You click the bookmark on any webpage you are auditing, and it injects a small, draggable, semi-transparent widget of our QA tracker in the corner of that page.
* **Why it's cool**: Instant to use, zero install, stays within "pure web".
* **Limitation**: You have to re-inject it if you reload or go to a new website, and some sites block it with security policies (CSP).
